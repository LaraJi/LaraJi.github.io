Harmonic Adversarial Attack Method

Posted on 2018-07-31 | Post modified: 2018-08-01 | In Paper | |

Arxiv Loc

Motivation

  • Previous adversarial samples generates noisy/edge-rich perturbations, which degrade the images quality;
  • In previous pixel-wise attacks, high visual quality and transferability has trade-off, can hardly be achieved at the same time;
  • harmonic function, using analytic complex functions as parametric model.

Method

harmonic function

Harmonic functions can generate edge-free perturbations from the perspectives of frequency domain analysis of images.

Harmonic function satisfies Laplace’s equation:
$$\frac{\partial^2f}{\partial x^2_1} + \frac{\partial^2f}{\partial x^2_2} + … + \frac{\partial^2f}{\partial x^2_n} = 0$$

  • Linear combination of harmonic functions is also a harmonic function;
  • Coordinate space affine transformation can argument the generated perturbations, including rotation, scaling, and translation;
  • gray-scale(each channel share the same harmonic perturbation across all three channels) or color harmonic perturbation(each channel has a separate harmonic perturbation);
  • Adversarial images generated using HAAM with specific harmonic functions can simulate some natural phenomena or photographic effects.

Experiment

Dataset: NIPS 2017 ‘Defense Against Adversarial Attack’

Metrics: visual quality(SSIM, Edge-SSIM) and transferability

  • Use Perturbation Norm Ratio (PNR), $PNR = \frac{||r(I)||}{||I||}$, to control the fairness of adversarial samples by different attacks in visual quality comparison;
  • Transferability comparison under similar visual quality.

Observations

  1. DeepFool and CWL2 can hardly generate adversarial samples with large PNR, and with high visual quality;
  2. Lower SSIM/ESSIM, higher transfer rates.

Discussion

  1. Defense: data augmentation?
  2. Help find corner cases for given models.
